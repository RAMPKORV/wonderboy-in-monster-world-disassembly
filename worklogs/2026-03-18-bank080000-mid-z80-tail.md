# 2026-03-18 bank080000 mid Z80 tail

- Split the former monolithic `0x098000-0x0991FF` Z80-like front slice in
  `src/bank080000_mid.asm` into the new ROM-order module `src/bank080000_mid_z80.asm`.
- Kept `0x098000-0x09907A` grouped as the opcode-dense Z80-like body because the loader and
  control flow are still unproven.
- Promoted the final `0x185` bytes into explicit source instead of keeping them inside the same
  large `incbin`: `0x09907B-0x09913C` is now an odd-aligned little-endian ascending word table,
  `0x09913D-0x0991FE` is a matching descending word table, and `0x0991FF` is a trailing zero
  byte before the already split `0x099200+` pre-descriptor data.
- The new names stay structural. The byte pattern proves lookup-table ownership and odd-aligned
  little-endian encoding, but it does not yet prove whether the values drive audio, motion,
  camera, or another Z80-side subsystem.
